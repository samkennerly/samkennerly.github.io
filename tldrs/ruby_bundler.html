ruby bundler

abstract

Bundler is a Ruby gem which installs Ruby gems.

Bundler is often used to install a unique bundle of gems for each software project on a machine. Keeping projects' gems separated helps developers avoid dependency hell.

basics

  1. Open a terminal and cd to a project's top-level folder.
  2. Create a Gemfile listing the project's required gems and version constraints.
  3. Run bundle to install all gems in the Gemfile and their dependencies.
  4. Bundler creates a Gemfile.lock file listing pinned versions of all installed gems.
  5. Bundler automatically updates Gemfile.lock on subsequent runs.

commands

The rake gem is used here as an example.
install Bundler gem install bundler
uninstall Bundler gem uninstall bundler
update Bundler gem update bundler
create a minimal Gemfile bundle init
install all gems in Gemfile bundle install
remove unused gems bundle clean
update rake bundle update rake
show path to rake bundle show rake
start interactive Ruby bundle exec irb
show installed gems bundle exec gem list
run a Ruby script bundle exec ruby path/to/script.rb
track gems with git git add Gemfile Gemfile.lock

dependencies

  • Ruby >= 2.3.0
  • RubyGems >= 2.5.0

examples

Example Gemfile file: 
ruby '~> 2.3'
source 'https://rubygems.org' do
    gem 'nokogiri'
    gem 'rails', '3.0.0.beta3'
    gem 'rack',  '>=1.0'
    gem 'thin',  '~>1.1'
end
Example .gitignore file: 
.bundle/           # Bundler config files
vendor/bundle      # Locally-installed gem files
